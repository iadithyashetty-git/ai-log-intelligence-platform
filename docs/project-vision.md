# Project Vision

## Project Name

AI Log Intelligence Platform

## One-Line Description

A local-first AI assistant that helps engineers understand application logs, failed transactions, and performance issues.

## Main Use Case

Given a transaction ID or log file, the system should explain what happened, where the failure occurred, which service was involved, and what the developer should check next.

## Why This Project Is Useful

This project connects my frontend experience with real AI engineering concepts such as LLMs, RAG, embeddings, ML classification, evaluation, and production AI UX.

## Final Demo Goal

The final demo should show:

1. Uploading logs
2. Filtering by transaction ID
3. Viewing transaction timeline
4. Asking AI what went wrong
5. Showing evidence from logs
6. Showing performance trends
7. Generating an incident report
